## TASK DESCRIPTION-

As part of the CODTECH internship program, Task-2 required the development of a **Weather Application** using a public API. The primary objective of this task was to design and implement a responsive web application that fetches and displays real-time weather data for any given city. The application needed to integrate with the **OpenWeatherMap API**, which provides accurate and up-to-date weather information. This project was not only about building a functional app but also about demonstrating professional coding practices, clean documentation, and user-friendly design.

The Weather Application is designed to allow users to input the name of a city and instantly retrieve weather details such as temperature, humidity, wind speed, and general conditions (e.g., sunny, cloudy, rainy). By leveraging the OpenWeatherMap API, the app ensures that the data presented is reliable and globally accessible. The task emphasizes the importance of API integration, error handling, and responsive design, which are critical skills for modern web developers.

From a technical perspective, the project is built using **HTML, CSS, and JavaScript**. HTML provides the structure of the application, CSS ensures a visually appealing and responsive layout, and JavaScript handles the logic of fetching data from the API and dynamically updating the user interface. The integration with OpenWeatherMap is achieved through the `fetch()` method in JavaScript, which allows asynchronous requests to the API endpoint. This ensures that the application remains responsive and provides a smooth user experience without unnecessary page reloads.

The application also incorporates **error handling mechanisms**. For example, if a user enters an invalid city name or if the API key is not valid, the app displays a clear error message instead of breaking. This demonstrates attention to detail and ensures that the application is robust and user-friendly. Additionally, the design is responsive, meaning it adapts seamlessly to different screen sizes, whether accessed on a desktop, tablet, or mobile device. This responsiveness is achieved through CSS media queries and flexible layouts.

This documentation ensures that anyone reviewing the project can quickly understand its purpose, functionality, and usage.
